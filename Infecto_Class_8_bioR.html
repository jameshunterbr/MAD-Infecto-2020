Normal String Functions with bioseq

  • Sequence is just a normal character vector

    • Can be manipulated with stringr functions

  • Special string functions (aware of limited alphabet)

    • seq_detect_pattern() - find a specific set of letters in a sequence

    • seq_remove_pattern() - find and remove a substring

    • seq_replace_pattern() - substitute one substring for another

    • seq_disambiguate_IUPAC() - show all alternatives for ambiguous IUPAC characters

Clustering Approach

  • Instead, we will cluster the consensus sequences based on their similarity
  • This uses bioseq::seq_cluster()
    • Implements hierarchical clustering method
      • One of principal unsupervised machine learning methods
      • Calculates the multivariate distance between all pairs of sequences
      • Groups these into similar “clusters” of items
        • Which have a minimum (threshold) distance to the neighboring clusters

Code for Clustering

fra_data_con <- fra_data_con %>% 
  mutate(cluster = seq_cluster(consensus_seq, 
                               threshold = 0.001))

Time to Draw the Tree - 3 Steps

  1. Convert consensus sequences to DNAbin format
    • bit-level encoding of sequences
    • binary encoding format permits more rapid comparisons of sequences
    • more economic storage
    • bioseq::as_DNAbin()
  2. Calculate distance matrix among all taxa
    • Call ape::dist.dna()
  3. Calculate a neighbor-joining tree with ape::bionj()

Code for Tree Calculation

fra_tree <- fra_data_con_2 %>% 
  as_DNAbin(consensus_seq, taxa_group) %>% 
  ape::dist.dna() %>%
  ape::bionj()

Draw with plot()

plot(fra_tree)

Heatmaps

  • Show visually relative strength of levels of 2 variables
  • Functions in base R (heatmap()) and ggplot (geom_tile())

Opinion of R Graph Gallery

The heatmap() function is natively provided in R. It produces high quality matrix and offers statistical tools to normalize input data, run clustering algorithm and visualize the result with dendrograms. It is one of the very rare cases where I prefer base R to ggplot2.

Advantages of Base R heatmap()

  • Can normalize data right within the function using scale = argument
  • Produces a dendrogram of clusters of items on both axes
  • Disadvantage: uses a matrix that needs row names
    • Row names no longer exist with tibbles
    • We need to recreate them

Fix Original Tibble (Rectangular Version)

  • The one from Excel
  • Turn tibble back into a data.frame and create row names
